# Instruções para o Modelo de IA (FrameBox)

Este arquivo contém as regras e diretrizes que você deve seguir ao trabalhar neste projeto para garantir a consistência e o sucesso da implementação.

## 1. Contexto do Projeto
O FrameBox é um NVR (Network Video Recorder) self-hosted para câmeras IP iCSee/Yoosee. As câmeras usam o protocolo proprietário **DVRIP** (porta 34567), não RTSP. Usamos o **go2rtc** como middleware para converter DVRIP → RTSP, e o **FFmpeg** para gravar e gerar HLS.

**Fluxo de dados:**
```
Câmera (DVRIP:34567) → go2rtc → RTSP local (:8554) → FFmpeg → HLS + MP4
```

## 2. Dados da Câmera de Teste (Validados)
- IP: `192.168.1.182`
- Porta: `34567` (DVRIP)
- Usuário: `<CAMERA_USER>`
- Senha: `<CAMERA_PASS>`
- Codec vídeo: **H265 (HEVC)**
- Codec áudio: **PCMA (G.711 A-law)**
- URL go2rtc: `dvrip://<CAMERA_USER>:<CAMERA_PASS>@192.168.1.182:34567?channel=0&subtype=0`

## 3. Princípios de Codificação
- **TypeScript em tudo**: Use tipos estritos e interfaces claras.
- **Modularidade**: Mantenha a lógica de go2rtc, FFmpeg e API separadas.
- **Performance**: Minimize o uso de CPU. Use `codec copy` no FFmpeg (sem re-encode).
- **Segurança**: Senhas devem ser hasheadas com `bcrypt`. Nunca expor URLs DVRIP/RTSP no front-end.
- **Resiliência**: Wi-Fi das câmeras é instável. Implementar watchdog com backoff exponencial.

## 4. Fluxo de Trabalho
Siga rigorosamente a ordem definida em `TASKS.md`. Não pule etapas sem concluir a anterior.

## 5. Estrutura de Pastas Esperada
Respeite a estrutura definida no `implementation_plan.md`:
- `go2rtc/` para o binário e config YAML do go2rtc.
- `backend/src/services/` para go2rtc-manager, stream-manager, storage-cleanup.
- `backend/src/modules/` para controllers e rotas da API.
- `mobile/` para o projeto Expo.

## 6. Ferramentas e Bibliotecas
- Backend: `express`, `better-sqlite3`, `jsonwebtoken`, `ffmpeg-static`.
- Mobile: `expo`, `react-native-video`, `axios`.
- Middleware: **go2rtc** (binário externo, gerenciado como child process).
- Linter e Formatação: **Biome** (configuração unificada).

## 7. Feedback e Verificação
Após cada task importante (ex: finalizar o Go2rtcManager), valide:
- Se o go2rtc está gerando o YAML correto.
- Se o FFmpeg está consumindo o RTSP do go2rtc e gerando arquivos no diretório `storage/`.
- Se a API está respondendo com os dados esperados.
